Висновки щодо результатів тестування API JSONPlaceholder

Тестовий план
Для тестування API JSONPlaceholder був розроблений тестовий план з використанням інструмента Apache JMeter. В тестовому плані були виконані GET, POST, PUT, PATCH та DELETE запити до API протягом 10 секунд у 3 ітерації.

Для кожного типу запиту було застосовано наступні Assertions:

1.Response Assertion:

Використовувався для перевірки наявності певного тексту в тілі відповіді.
Допоміг перевірити, чи повертає сервер очікувані дані.

2.Size Assertion:

Використовувався для перевірки розміру відповіді.

3.Duration Assertion:

Використовувався для перевірки часу відповіді на запит.

Результати тестування

View Results in Tree
було зафіксовано наступні результати для кожного типу запиту:

GET: Всі GET запити повернули очікувані дані. Всі Assertion для GET запитів пройшли успішно.

POST: Всі POST запити були успішно виконані і повернули статус 201. Розміри відповідей відповідали очікуваному.

PUT: PUT запити повернули очікувані дані. Розмір відповіді та час відповіді були в межах припустимих значень.

PATCH: PATCH запити були успішними, але було виявлено деякі відхилення у розмірах відповідей.

DELETE: DELETE запити успішно видалили ресурси. Час відповіді був в межах припустимих значень.

Summary Report
було отримано наступні підсумкові дані:

Загальна кількість запитів: 15 (5 запитів для кожного типу)
Середній час відповіді для всіх запитів склав: 1,1/sec.


Висновки
Тестування API JSONPlaceholder показало, що більшість запитів були успішно виконані та повернули очікувані дані.
